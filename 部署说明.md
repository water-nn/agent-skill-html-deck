# 部署说明

远程访问链接：

https://water-nn.github.io/agent-skill-html-deck/

GitHub 仓库：

https://github.com/water-nn/agent-skill-html-deck

## 1. 本地如何双击预览

在项目文件夹里双击：

`打开预览.cmd`

它会自动启动本地服务并打开浏览器。

请不要关闭弹出的黑色命令窗口。关闭后，浏览器里的本地预览也会打不开。

## 2. 如何修改内容

主要改这个文件：

`src/slides.tsx`

标题、正文、卡片、列表、每一页内容都优先在这里修改。

更详细的新手说明请看：

`如何修改内容.md`

## 3. 如何重新构建

如果你想确认项目能正常打包，在终端进入项目目录后运行：

```bash
npm run build
```

看到 `✓ built` 就表示构建通过。

## 4. 如何推送后自动更新 GitHub Pages

项目已经配置 GitHub Pages workflow：

`.github/workflows/deploy.yml`

以后改完内容后，按顺序运行：

```bash
git add .
git commit -m "Update presentation content"
git push
```

推送到 GitHub 的 `main` 分支后，GitHub Actions 会自动构建并更新远程页面。

## 6. 如果页面空白

优先检查：

`vite.config.ts`

本项目部署在 GitHub Pages 的项目仓库路径下，所以 `base` 应为：

```ts
base: '/agent-skill-html-deck/'
```

如果仓库名变了，这里的路径也要跟着变。

## 7. 如果 Actions 失败

去这里看错误：

GitHub 仓库页面 → Actions → Deploy HTML Presentation Deck

常见检查点：

- `npm ci` 是否安装依赖失败
- `npm run build` 是否构建失败
- `.github/workflows/deploy.yml` 是否被误改
- GitHub Pages Source 是否是 `GitHub Actions`
