# 部署说明

当前项目已经可以本地双击预览，也已经准备了 GitHub Pages workflow 草稿：

`.github/workflows/deploy.yml`

## 当前远程部署条件

我检查了当前 git 状态：这个项目位于一个本地 git 工作区中，但没有检测到 GitHub remote。

因此现在还不能直接得到远程访问链接。

缺少的条件是：

1. 需要把项目放到一个 GitHub 仓库里
2. 需要设置 `git remote`
3. 需要把代码推送到 GitHub
4. 需要在 GitHub 仓库设置里启用 Pages，来源选择 GitHub Actions

## 方案 A：GitHub Pages

如果仓库地址类似：

`https://github.com/<GitHub用户名>/<仓库名>`

最终链接通常是：

`https://<GitHub用户名>.github.io/<仓库名>/`

本项目的 `vite.config.ts` 已经支持 GitHub Actions 环境下自动设置 Vite `base`，通常不需要手动改。

部署前请确认：

```bash
git remote -v
```

能看到 GitHub 地址。

## 方案 B：Vercel

如果你想用 Vercel：

1. 先安装 Vercel CLI：

```bash
npm install -g vercel
```

2. 登录：

```bash
vercel login
```

3. 在项目目录运行：

```bash
vercel
```

如果部署成功，把 Vercel 输出的链接保存到：

`DEPLOY_URL.txt`

## 方案 C：只保留部署说明

当前就是这个状态：本地项目已经准备好，但没有自动部署，也没有远程链接。
