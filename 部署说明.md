# 部署说明

当前项目对应的 GitHub 仓库：

`https://github.com/water-nn/agent-skill-html-deck`

当前远程访问链接：

`https://water-nn.github.io/agent-skill-html-deck/`

远程链接也已写入：

`REMOTE_URL.txt`

## 1. 本地如何双击预览

双击项目根目录里的：

`打开预览.cmd`

它会自动启动本地服务并打开浏览器。

注意：预览时会出现一个黑色命令窗口。这个窗口不要关闭，关闭后网页预览也会停止。

## 2. 如何修改内容

主要修改：

`src/slides.tsx`

这里放着每一页的标题、正文、卡片、标签和示例内容。

更详细的新手说明见：

`如何修改内容.md`

## 3. 如何重新构建

在项目目录运行：

`npm run build`

构建结果会生成到：

`dist`

## 4. 如何推送后自动更新 GitHub Pages

项目已经准备了 GitHub Pages workflow：

`.github/workflows/deploy.yml`

把代码提交并推送到 GitHub 的 `main` 分支后，GitHub Actions 会自动执行：

1. checkout
2. setup-node
3. npm ci
4. npm run build
5. upload-pages-artifact，路径为 `dist`
6. deploy-pages

## 5. 最终远程访问链接在哪里看

优先看：

`REMOTE_URL.txt`

当前链接是：

`https://water-nn.github.io/agent-skill-html-deck/`

也可以在 GitHub 仓库页面查看：

Settings → Pages

## 6. 如果页面空白，优先检查什么

优先检查：

`vite.config.ts`

这个项目部署到项目仓库 GitHub Pages，因此 `base` 应为：

```ts
base: "/agent-skill-html-deck/"
```

如果以后部署到用户名根站点，例如 `water-nn.github.io` 这个特殊仓库，才改成 `/`。

## 7. 如果 GitHub Actions 失败，去哪里看错误

打开 GitHub 仓库：

`https://github.com/water-nn/agent-skill-html-deck`

然后进入：

Actions → 选择失败的工作流 → 展开红色失败步骤

常见问题：

- `npm ci` 失败：检查 `package-lock.json` 是否提交
- `npm run build` 失败：先在本地运行 build 修复
- 页面空白：检查 `vite.config.ts` 的 `base`
- Pages 没启用：Settings → Pages → Build and deployment → Source → GitHub Actions
